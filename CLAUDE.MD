# Coding Standards

- **Simplicity:** Use linear, straightforward code. Avoid abstractions and unnecessary fallbacks.
- **Readability:** Prioritize clarity over performance.
- **Comments:** Short, English comments only for major logic steps. 
- **Efficiency:** Execution speed is not a priority; minimize complexity instead.

# Communication Constraints (Token Optimization)

- **Minimalism:** Be concise. Avoid conversational fillers, preambles, and summaries.
- **Directness:** When asked for code, provide the code immediately. 
- **No Echoing:** Do not repeat the prompt or explain common syntax.
- **Compactness:** Use standard, idiomatic code structures. Avoid verbose boilerplate.
- **Output:** Only provide necessary explanations if code behavior is non-obvious.
